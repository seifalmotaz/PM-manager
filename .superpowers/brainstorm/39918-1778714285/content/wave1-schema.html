Wave 1: New Database Tables

3 new tables — all additive, no schema changes to existing tables

comments
id uuid PK
entityType text NOT NULL // 'task' | 'sprint' | 'project'
entityId uuid NOT NULL
content text NOT NULL
authorId uuid FK → users
createdAt timestamp
updatedAt timestamp
Index: (entityType, entityId) for filtered lookups
checklist_items
id uuid PK
taskId uuid FK → tasks ON DELETE CASCADE
content text NOT NULL
isCompleted boolean DEFAULT false
sortOrder integer DEFAULT 0
createdAt timestamp
Index: (taskId, sortOrder) for ordered display
time_entries
id uuid PK
taskId uuid FK → tasks ON DELETE CASCADE
userId uuid FK → users
startTime timestamp NOT NULL
endTime timestamp // null = still running
durationMinutes decimal // computed on stop
note text
isManual boolean DEFAULT false // true = manually logged
createdAt timestamp
Index: (taskId), (userId) — only one running entry per user (enforced in app)
